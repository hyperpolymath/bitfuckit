= Contributing to bitfuckit
:toc:
:toclevels: 2

Thank you for your interest in contributing to bitfuckit - the community-built
Bitbucket CLI that Atlassian never made!

== Language Policy

bitfuckit follows the hyperpolymath https://github.com/hyperpolymath/rhodium-standard-repositories[Rhodium Standard]:

[cols="1,2,2"]
|===
|Language |Use Case |Notes

|*Ada 2012* |Primary application code |SPARK verified where possible
|*Nickel* |Configuration |For complex config (config.ncl)
|*Bash/POSIX* |Scripts, automation |Keep minimal
|*Guile Scheme* |State/meta files |STATE.scm, META.scm, etc.
|===

*BANNED*: TypeScript, Node.js, npm, Go, Python (except SaltStack)

== Getting Started

. Fork the repository
. Clone your fork:
+
[source,bash]
----
git clone https://github.com/YOUR_USERNAME/bitfuckit.git
cd bitfuckit
----

. Install dependencies:
+
[source,bash]
----
# Fedora
sudo dnf install gcc-gnat gprbuild

# Debian/Ubuntu
sudo apt install gnat gprbuild
----

. Build:
+
[source,bash]
----
gprbuild -P bitfuckit.gpr
----

== Code Style

* Ada 2012 style guide
* 3-space indentation
* SPDX license header on all source files
* Use SPARK Mode where possible for specification files

== Submitting Changes

. Create a feature branch:
+
[source,bash]
----
git checkout -b feature/my-feature
----

. Make your changes
. Ensure the build succeeds:
+
[source,bash]
----
gprbuild -P bitfuckit.gpr
bin/bitfuckit --help
----

. Commit with a descriptive message
. Push and create a pull request

== Adding New Commands

. Add the command handling in `src/bitfuckit.adb`
. If API calls needed, add to `src/bitbucket_api.adb`
. Update the help text in `Print_Usage`
. Add shell completions in `completions/`
. Update the man page in `doc/bitfuckit.1`

== Questions?

Open an issue at https://github.com/hyperpolymath/bitfuckit/issues
